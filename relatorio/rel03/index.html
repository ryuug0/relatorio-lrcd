1 O que é o LEEM?

O Laboratório de Ensino de Estatística e Matemática (LEEM) é um pacote educacional para matemática e estatística tanto a nível escolar quanto superior. Desenvolvido pelo professor da atual disciplina, Ben Dêivide, tem como objetivo auxiliar no processo de ensino e aprendizagem do conteúdo no ambiente R.

2 Como Utilizar?

O pacote LEEM precisa de um interpretador R para funcionar e, para auxiliar no desenvolvimento dos códigos, é de suma importância a utilização de uma IDE. No nosso caso, utilizaremos o RStudio.

2.1 Ambiente de Desenvolvimento R

  • O arquivo de instalação do interpretador R pode ser encontrado no seguinte link .

  • Já o link para download do RStudio pode ser acessado clicando aqui

2.2 Instalação das Bibliotecas

Com o R e o RStudio instalados, precisamos adicionar a biblioteca ao ambiente. Atualmente, existem dois métodos para isso:

1. Via CRAN

Para isso, basta a seguinte linha de comando dentro do RStudio:

install.packages("leem")

2. Via GitHub (recomendável)

A instalação via GitHub necessita da instalação de outras bibliotecas dependentes. A seguir os códigos que devem ser rodados linha por linha

# Instalar os pacotes dependentes
pkgs <- c("manipulate", "tkRplotR", "tkrplot", "crayon", "diagram")
install.packages(pkgs)
# install.packages("devtools")
devtools::install_github("bendeivide/leem")

As bibliotecas dependentes instaladas são:

  • devtools: Ferramentas de desenvolvimento
  • tkrplot e tkRplotR: Exibição de gráficos em janelas externas
  • manipulate: Interatividade em tempo real
  • crayon: Estilização de terminal
  • diagram: Construção de diagramas e fluxogramas
Nota

A biblioteca tcltk geralmente é instalada junto com o R. Caso ela não esteja no sistema, é necessário instalá-la para o funcionamento das interfaces gráficas.

2.3 Importação do LEEM Para o Ambiente R

Com o ambiente e as bibliotecas instalados, basta importar a biblioteca LEEM para o ambiente através da seguinte linha de código:

library(leem)

Agora a biblioteca está pronta para uso, podendo ser testada com o comando para abrir sua interface gráfica:

leem()

3 Funções

A seguir, serão apresentadas as principais funções e utilidades do pacote LEEM

3.1 Tabulação de Dados

A tabulação consiste em resumir um conjunto de dados brutos em uma tabela de classes com suas devidas frequências. Isso facilita a análise humana: no exemplo a seguir, ao invés de avaliar um vetor com 1000 números isolados, analisaremos uma tabela com apenas 3 faixas de valores, suas frequências e a amplitude total da amostra.

Na biblioteca LEEM, utiliza-se a função new_leem para transformar um vetor em um objeto do tipo leem. Essa função recebe o parâmetro variable, que define a natureza do dado (1 para qualitativa e 2 para quantitativa). Em seguida, a função tabfreq utiliza o parâmetro k para definir o número de classes que a tabela terá.

Código 
library(leem)
set.seed(10)
dados <- rnorm(1000, mean = 100, sd = 2)
tab <- dados |>
  new_leem(variable = 2) |>
  tabfreq(k = 3); tab

Table of frequency 
Type of variable: continuous

             Classes  Fi      PM   Fr Fac1 Fac2 Fp Fac1p Fac2p
1   90.7 |---  97.25  89  93.975 0.09   89 1000  9   8.9 100.0
2  97.25 |---  103.8 882 100.525 0.88  971  911 88  97.1  91.1
3 103.8 |---  110.35  29 107.075 0.03 1000   29  3 100.0   2.9

============================================== 
Classes: Grouping of classes 
Fi: Absolute frequency 
PM: Midpoint 
Fr: Relative frequency 
Fac1: Cumulative frequency (below) 
Fac2: Cumulative frequency (above) 
Fp: Percentage frequency 
Fac1p: Cumulative percentage frequency (below) 
Fac2p: Cumulative percentage frequency (above)

Interpretando saídas:

  • Classes: São os intervalos de agrupamento dos dados.
  • Fi: Frequência absoluta, ou seja, quantos números estão dentro da Classe
  • PM: Ponto médio, o valor central da classe.
  • Fr: Frequência relativa, representa a proporção de cada classe em relação ao total.
  • Fac1 / Fac2: Frequências acumuladas (“abaixo de” e “acima de”, respectivamente).
  • Fp / Fac1p / Fac2p: Representação das Frequências em formato de porcentagem (%).

3.2 Apresentações Gráficas

O pacote conta com apresentações gráficas próprias para lidar com os dados tabulados, facilitando ainda mais a visualização destes.

3.2.1 Histogramas

Os Histogramas são utilizados para visualizar a distribuição de dados quantitativos. No LEEM, o gráfico é gerado através da função hist(), que interpreta automaticamente as classes e frequências calculadas no objeto leem.

Código 
tab |>
  hist(
    freq = "p", # tipo da frequência
    bg = FALSE,  # Sem/com fundo
    barcol = c("blue","purple","yellow"), # Cores nas barras
    xlab = "Variável X"
  )

Principais Parâmetros

  • freq: Define o tipo da frequência mostrado
    • “a” para absoluto(default)
    • “r” para relativo
    • “p” para porcentagem
  • main: Título do gráfico
  • xlab e ylab: Nome dos eixos x e y, respectivamente

3.2.2 Ogivas

As Ogivas são utilizadas para visualizar as frequências acumuladas de um conjunto de dados. No LEEM a função ogive() que plota tanto o acúmulo “abaixo” quanto o “acima”, a mediana pode ser observada pelo cruzamento dessas linhas.

Código 
tab |>
  ogive(
    bars = TRUE, # Inserir barras de fac1/2
    both = TRUE,
    barcol = c("yellow", "purple"), # cor da barra
    lpcol = "blue", # cor da linha
    pch = 8 # Tipo do ponto
  ) |>
  insert(
    type = "median",
    lcol = "red",
    ptext = 0.15
  )

Principais Parâmetros

  • bars: Insere as barras de frequência acumulada 1 e 2
  • both: Exibe as ogivas simultaneamente, mostrando a mediana
  • decreasing: Define a direção da curva, crescente ou decrescente

3.2.3 Gráfico de Pizza

O Gráfico de Pizza é utilizado para visualizar a composição proporcional das classes em relação ao todo. As frequências relativas do objeto são extraídas pela função piechart()

Código 
tab |>
piechart(
  col = c("blue","purple","yellow"),
  border = TRUE,
)

Principais Parâmetros -col: Define as cores das fatias -border: Insere bordas entre as fatias -labels: Permite renomear as legendas das fatias

3.3 Medidas de Posição

As Medidas de Posição (média, moda e mediana) podem ser obtidas através de uma única função no LEEM: a mpos(). Isso facilita e agiliza a análise dos dados, além de ser integrada com os dados tabulados do objeto leem

Código 
tab |>
  mpos(grouped = FALSE)
$average
[1] 100.02

$median
[1] 99.99

$mode
[1] "The data set has no mode!"

Principais Parâmetros

  • grouped: Alterna entre o cálculo dos dados brutos (False) e dados tabulados leem (True)
  • rounding: Define o número de casas decimais
  • details: Exibe as fórmulas e passo a passo (ideal para fins didáticos)

3.4 Medidas de Dispersão

As Medidas de Dispersão medem o grau de variabilidade dos dados, ou seja, o tanto que a amostra está afastada da média central. Podem ser medidas de 3 formas utilizando o LEEM, que serão abordadas a seguir.

  • Variância: Mede o quão longe os valores da amostra estão da média aritmética através da função variance()
Código 
tab |> variance()
[1] 4.912963
  • Desvio Padrão: É a raiz quadrada da variância, indica a dispersão dos dados na mesma unidade de medida da amostra original com sdev().
Código 
tab |> sdev()
[1] 2.22
  • Coeficiente de Variação: Mede a variabilidade relativa em relação à média, expressa em porcentagem com a função cv()
Código 
tab |> cv()
[1] 2.22

Principais Parâmetros

Os três possuem parâmetros principais iguais, são eles:

  • grouped: Alterna entre o cálculo dos dados brutos (False) e dados tabulados leem (True)
  • rounding: Define o número de casas decimais
  • details: Exibe as fórmulas e passo a passo (ideal para fins didáticos)

3.5 Distribuição e Probabilidades

Os cálculos de Probabilidade no LEEM são feitos através da função P(). Essa função aceita intervalos como argumento, além de chamar o tcltk para gerar uma interface própria.

  • Distribuição Normal - Probabilidade em um Intervalo:
Código 
P(-1 %<=X<=% 1, mean = 0, sd = 1)
[1] 0.68269
  • Distribuição Binomial - Probabilidade Pontual:
Código 
P(5, dist = "binomial", size = 10, prob = 0.5)
[1] 0.62305

Principais Parâmetros:

  • q: Define o ponto ou o intervalo de interesse
  • dist Especifica a distribuição desejada
    • “normal”(default)
    • “binomial”
    • “t-student”
    • “chisq”
    • “f”
  • lower.tail: Define se a probabilidade será calculada à esquerda (TRUE) ou à direita (False)
  • porcentage: Alterna entre o resultado em decimal ou em porcentagem
  • gui: Define a interface de saída
    • “plot”: Gera um gráfico estático no RStudio
    • “tcltk”: Abre uma janela interativa

3.6 Teste de Hipóteses

O Teste de Hipóteses serve para verificar se uma suposição sobre a população é estatisticamente viável com base em uma amostra. No LEEM, a função th() automatiza o cálculo e pode gerar os gráficos das regiões de aceitação e rejeição (região crítica).

Código 
h0 <- 90
set.seed(10)
data <- rnorm(30, 100, 10)
th(data, h0 = h0, sd = 10, plot = TRUE)


   One Sample z-test (Two-sided test) 
 
Step 1: Hypothesis 
 H0: mu = 90 
 H1: mu != 90 

Step 2: Significance level 
 alpha = 0.05 

Step 3: Rule of decision 
   If |ztest| > |ztab| => Reject H0! 
   ztest - test statistic; ztab - critical point 
So... 
As ztest = 3.59 > |ztab =1.96 then reject H0! 
Otherwise... 
As p-value =  0.00033  < alpha =  0.05   then reject H0! 

Step 4: Conclusion 
We observed by the Z Test that the null hypothesis was rejected, at the significance level of   5 % probability

Principais Parâmetros:

  • test: Define o tipo de teste
    • “ztest”
    • “ttest”
    • “ptest”
    • “anova”
  • h0: Define o valor do parâmetro para a hipótese nula
  • alternative: Define a hipótese alternativa
    • “two.sided”: bilateral
    • “less”: unilateral inferior
    • “greater”: unilateral superior
  • alpha: Nível de significânica
  • plot: Gera o gráfico

4 Conclusão

O LEEM ajuda a aplicar os conceitos estatísticos no R de um jeito fluido. Ele agiliza a criação de tabelas e gráficos, permitindo que o foco seja a análise e interpretação dos resultados. Por possuir funções intuitivas, uma documentação completa e uma curva de aprendizado relativamente baixa, o pacote atende bem tanto quem está começando agora quanto quem busca realizar análises mais avançadas de forma direta.
